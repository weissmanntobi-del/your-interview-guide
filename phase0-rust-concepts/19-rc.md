# Concept 19: `Rc<T>` — Reference Counting

## Plain English

`Rc<T>` stands for **Reference Counted**. It lets **multiple parts of your code own the same value** — the value is only dropped when the last owner is gone.

Normal Rust ownership: one value, one owner. `Rc<T>` breaks that rule — safely — by keeping a count of how many owners exist.

**Important:** `Rc<T>` is for **single-threaded** code only. For multiple threads, use `Arc<T>`.

---

## Code Example

```rust
use std::rc::Rc;

fn main() {
    let a = Rc::new(String::from("hello"));
    let b = Rc::clone(&a);  // b is another owner — count is now 2
    let c = Rc::clone(&a);  // c is another owner — count is now 3

    println!("{}", Rc::strong_count(&a)); // 3
    // value is dropped only when a, b, AND c are all gone
}
```

## Mini Quiz

**Q1.** What problem does `Rc<T>` solve?
- A) It allows a value to have multiple owners in single-threaded code ✓
- B) It makes values thread-safe

**Q2.** When is the value inside an `Rc<T>` dropped?
- A) When the first clone is dropped
- B) When all clones (all owners) are dropped ✓

---

## Quiz Answers

**Q1 → A.** `Rc<T>` enables multiple ownership in single-threaded code by reference counting. For thread safety, use `Arc<T>`.

**Q2 → B.** The value is dropped only when the reference count reaches zero — meaning all owners (`Rc::clone`s) have been dropped.
